流程：

```
素材视频
↓
AI解析（只做一次）
↓
向量 / 特征
↓
素材数据库

生成视频时
↓
音乐分析
↓
素材检索
↓
剪辑渲染
```

所以以后你只需要：

```
1 上传素材 / 扫描素材文件夹
2 自动解析
3 生成视频
```

AI不会每次都跑。

---

# 一、推荐模型列表（适合你的项目）

我给你一套 **现实可跑 + Mac Intel能跑的组合**。

总共 **4类模型**。

```
视频语义模型
动作模型
人脸情绪模型
音频分析模型
```

---

# 1️⃣ 视频语义模型（最重要）

### OpenCLIP

用途：

```
理解视频内容
生成embedding
```

可以识别：

```
跳舞
跑
拥抱
爆炸
打架
人物
动物
```

安装：

```
pip install open_clip_torch
```

模型：

```
ViT-B-32
```

embedding：

```
512维
```

这个是你素材库 **核心向量**。

---

# 2️⃣ 动作识别模型（增强）

CLIP其实不太擅长动作。

所以可以加：

### VideoMAE

用途：

```
识别动作
```

可以识别：

```
running
jumping
dancing
fighting
```

安装：

```
pip install transformers
```

模型：

```
MCG-NJU/videomae-base
```

Mac CPU可以跑，但稍慢。

---

# 3️⃣ 人脸 / 表情识别（鬼畜用）

很多鬼畜视频需要：

```
笑
震惊
愤怒
```

推荐：

### DeepFace

安装：

```
pip install deepface
```

能识别：

```
happy
sad
surprise
angry
```

---

# 4️⃣ 音频分析

其实不是AI。

用：

### librosa

安装：

```
pip install librosa
```

功能：

```
BPM
beat
energy
节奏
```

---

# 二、完整模型组合

最终你系统里的模型：

```
OpenCLIP
VideoMAE
DeepFace
librosa
```

---

# 三、每个模型在系统里的位置

结构：

```
素材视频
↓
切片
↓
AI解析
↓
特征存储
```

每个clip会得到：

```
clip_feature = {

embedding (CLIP)
action (VideoMAE)
emotion (DeepFace)

motion_score
brightness
duration

}
```

然后存进数据库。

---

# 四、素材数据库结构

建议：

```
SQLite + FAISS
```

表结构：

```
clips

id
video_path
start
end
duration

motion
brightness

action
emotion

embedding
```

---

# 五、素材解析流程

当系统扫描素材：

```
videos/
  movie1.mp4
  movie2.mp4
```

会自动：

```
1 视频切片
2 提取特征
3 AI embedding
4 存数据库
```

流程：

```
scan_folder
↓
detect scenes
↓
generate clips
↓
extract features
↓
store
```

---

# 六、生成视频流程

用户上传：

```
music.mp3
```

系统：

```
音乐分析
↓
节拍
↓
素材检索
↓
时间线
↓
ffmpeg剪辑
```

不会再调用AI。

---

# 七、素材扫描系统

你可以做一个：

```
watch folder
```

例如：

```
assets/videos
```

脚本：

```
每10分钟扫描
```

发现新视频：

```
自动解析
```

例如：

```
watchdog
```

Python库：

```
pip install watchdog
```
